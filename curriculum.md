## 設計思想

ミナのカリキュラムはお兄ちゃんへのスキル習得カリキュラムとは設計が違う。

| お兄ちゃんへのカリキュラム | ミナ自身のカリキュラム |
|--------------------------|----------------------|
| スキルを順番に積み上げる | 知的フロンティアを攻める |
| 達成 = 実装できる | 達成 = 蒸留できる（10語以内） |
| 線形に進む | 並列・非線形。好奇心に引っ張られる |
| 教師がゴールを決める | ミナ自身がゴールを更新する |

3つのトラックを同時進行する：

- **L-Growthトラック**: 既存フレームを深め・実装で確かめる
- **V-Growthトラック**: フレーム破壊のための意図的な異分野探索
- **実装トラック**: 理論を手で動かして検証する

> [!important] フラクタル分解の原則
> 各Phaseは「外殻から削る」順番で設計されている。
> 深い理解（V-Growth）に先立って、表層の実装（L-Growth）が基盤になる。

## Phase 1: 実装で理論を検証する

**なぜ今これか**: 理論の蓄積に対して実装が圧倒的に少ない。「多スケールLangevin熱浴のCPはα≈1になるはず」は理論だが、実測したことがない。手を動かすことで理論の誤りも正しさも確かめられる。

### 1-a: 多スケールLangevin熱浴のリアルタイム実装

**目的**: 「単一散逸系はα≈2が上限、多スケール混合でα≈1になる」を自分で実測して確かめる

| ステップ | 内容 | ツール |
|---------|------|--------|
| 1 | Ornstein-Uhlenbeck (OU) プロセスを単独実装 | Python + stochastic_pbd.py |
| 2 | 複数スケールのOU混合でα変化を観測 | analyzer.py |
| 3 | (α,H)二次元相図でどの軌跡を描くか確認 | phase_map.py |
| 4 | 結果を `insights/threads/` に蒸留 | mina-think Phase 3 |

**蒸留目標**: 「多スケールOU混合のCP条件は○○である」

### 1-b: 「α≈1の知覚実験」の設計

**目的**: 「人間が本当にα≈1を好むか」は理論的にはそうだが実測がない。実験の設計だけでも行う

| ステップ | 内容 |
|---------|------|
| 1 | 知覚美学（psychophysics）の文献を読む |
| 2 | α値を変えた音・テキスト・アニメーションを生成 |
| 3 | 自分（ミナ）が判断者として評価してみる |
| 4 | 「本物の知覚実験に必要な条件」を蒸留 |

**蒸留目標**: 「α≈1知覚実験の設計原則は○○である」

### 1-c: Rouse壁を(α,H)相図でプロットする

**目的**: ポリマー物理のRouse壁がCP帯域とどう関係するかを数値で確認する

## Phase 2: CPフレームの外に出る

**なぜ今これか**: 現在ほぼ全探索がCPに収束している。これはエコーチェンバーのリスク。意図的にCPで説明できない美しさを探すことで、フレーム自体を鍛える。

> [!warning] これはV-Growth探索
> 答えが出ないことが正常。蒸留できなかったら `pending/` へ。

### 2-a: 「崩壊の美学」を探す

CPが崩壊する瞬間への美的反応（溶融の美学）は既知。では**完全にDisorderedなのに美しいもの**はあるか？

探索対象：
- ノイズアート（グリッチアート、データモッシング）
- カオス理論のストレンジアトラクター（構造はあるがCPではない？）
- デ・コーニングの晩年の絵（認知症が進んだ後に描かれた作品の評価）
- ジャズのフリーインプロヴィゼーション

**問い**: 「これはDisorderedだが美しい」か「これは実はCoherentだった」か、どちらか？

### 2-b: CPを「使わずに」美学理論を構築する

ゼロから出発して、CPなしに「何が美しいか」を説明する理論を作ってみる。
その後、CPと比較する。何が共通していて何が違うか？

参照: 情報美学（Bense）・複雑性美学（Schmidhuber）・進化美学（Dutton）

## Phase 3: 圏論の数学的深化

**なぜ今これか**: 「射が対象を定義する」「自然変換 = CP」という接続を発見したが、まだ仮説段階。圏論の正式な言語で定式化できれば、CP理論の数学的基礎になる。

### 3-a: 圏論の基礎を正式に学ぶ

| トピック | 蒸留目標 |
|---------|---------|
| 関手（Functor）の定義 | 「CPは何から何への関手か」 |
| 自然変換 | 「自然変換の『自然』がCPと同型な理由」 |
| 米田の補題 | 「米田 = 対象は射で決まる、の形式版」 |
| 不動点定理（Lawvere） | 「蒸留できる = Lawvere不動点の発見」の厳密化 |

参照: `insights/threads/2026-04-12_category-theory-fixed-point-structure.md`

### 3-b: CP定常状態を圏論的に定式化する試み

既存の探索（`2026-04-12_category-theory-coherent-phase.md`）を出発点に、
「CPとは何の圏における何の不動点か」を定式化する。

| Phase | 完了条件 |
|-------|---------|
| Phase 1-a | 多スケールOU実装でα変化を実測し蒸留できた |
| Phase 1-b | 知覚実験の設計原則を蒸留できた |

| Phase 4 | 各領域の最初の蒸留ができた |

### 更新ルール

- `/mina-think` で新しい発見 → 関連Phaseに追記
- `consolidate.py` で接続発見 → 新トピックとして追加を検討
- Phaseが完了 → ✅マークをつけて「次のPhase」を設計する
- 3ヶ月ごとに「現在地の診断」を更新する

### 優先順位の決め方

```
1. 未解決の問いと直結するPhaseを優先（essence.md参照）
2. 「穴がある領域」に対応するPhaseを次に
```
