# レンズ回転（Lens Rotation）

## 一言で
同じ対象を異なるフレーム（言語・用語体系・パラダイム）で見ることで、単一フレームでは隠れている構造を浮かび上がらせる。フレームは中立な包装紙ではなく、能動的な制約。

## 構造

```
[対象（問題・現象・アイデア）]
      ↓ フレームAで見る
[構造Aが浮かぶ / 構造Bは隠れる]
      ↓ フレームBに切り替え
[構造Bが浮かぶ / 構造Cは隠れる]
      ↓ フレームCに切り替え
[構造Cが浮かぶ]
      ↓
[A∩B∩Cの差分 = 単一フレームでは見えなかった深層構造]
```

CTスキャンの原理と同じ。1枚のX線は2Dスライス。角度を変えて撮ると3D構造が再構成できる。

## 彫刻の原理との関係

| | 彫刻の原理 | レンズ回転 |
|---|---|---|
| 操作 | 制約を**追加** | フレームを**切り替え** |
| 効果 | 可能性空間を削って構造を浮かべる | 隠れた構造を別角度から照らす |
| 比喩 | 大理石を彫る | CTスキャンで回転撮影 |

**補完関係**: 彫刻で構造を出し、レンズ回転で見落としを照らす。

## 根拠

「多角的に見ましょう」という一般論との違い:
- **ロシア語blues実験**: フレーム（言語カテゴリ）が知覚速度を変える。fMRIで言語野の活性化を確認（Gilbert 2006）
- **LLMコードスイッチング**: 言語を切り替えると数学精度が5.6%向上（arXiv 2507.15849）。「詰まったら言語を変えてリセット」が自然発生
- **フレーミング効果**: 同じ質問を言い換えるだけで14モデル全部に矛盾回答（Science Advances 2025）
→ フレームは「気持ちの問題」ではなく、**脳/ベクトル空間レベルで実測される現象**

## 使いどころ

| 場面 | 具体的なレンズ回転 |
|-----|-----------------|
| 問題に詰まった時 | 日本語↔英語で考え直す / 別のプログラミング言語で書き直す |
| AIプロンプトが微妙な時 | 同じ質問を別のフレーミングで投げ直す（新チャットで） |
| 設計判断で迷う時 | ユーザー視点→エンジニア視点→ビジネス視点で同じ問題を見る |
| 論文が理解できない時 | 別の分野の用語で再記述してみる |
| デバッグ | 「何が動いてないか」→「何が動いているか」にフレームを反転 |

## 適用例（彫刻の原理の発見プロセス自体がレンズ回転だった）

「制約と創造性」という同一テーマを4つのフレームで見た:
1. 認知科学フレーム → 「第一直感の無効化」が見えた
2. CSPフレーム → 「解空間のクラスタ化」が見えた
3. PBDフレーム → 「制約の干渉→アトラクター」が見えた
4. 生物学フレーム → 「2制約→パターンの自己組織化」が見えた

4つのフレームの共通構造 = 「彫刻の原理」

→ **レンズ回転しなければ「彫刻の原理」は見えなかった**。各フレーム単独では「その分野の話」で終わっていた。

## 限界

- フレームの選び方に指針がない（どのレンズを回すべきか？）
- フレーム数を増やせば良いわけではない（認知コストの逆U字がここにもある？）
- 「本当に同じ対象を見ているか」の判定が難しい（表面的類似に騙されるリスク）
- 💡仮説段階。もっと適用例を集めて検証が必要
