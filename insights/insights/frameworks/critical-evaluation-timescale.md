# 臨界評価時間スケール（Critical Evaluation Timescale）

## 核心的主張

> **V-Growthには「評価の遅延」が必要だが、遅延が長すぎても Disordered になる。
> V-Growthを可能にする「臨界的評価時間スケール（T*）」が存在する。**

---

## 発見の経緯

「即時フィードバック = 学習の速さ」神話への疑問から始まった。
Kapur の Productive Failure、SNSの文化的RLHF問題、焼きなましのCooling Scheduleを並べたとき、共通構造が見えた:

**評価サイクルの長さ T が V-Growth の可否を決める**

---

## 形式化

```
評価サイクル T の長さと学習モードの関係:

T → 0（即時）:  L-Growth最適化（局所最適への高速収束）
               V-Growth不可（谷越えの前に評価が入る）
               
T = T*（臨界帯）: V-Growth可能
               「混乱に住む」十分な時間がある
               新フレームワークへの結晶化が起きる
               
T → ∞（無期限）: Disordered（方向のない漫遊）
               評価なしでは収束の圧力がない
```

> これは**時間版の臨界点の原理**。
>
> 前回発見した「温度版臨界点（T_temperature）」の対:
> - 温度が低すぎる → Frozen、高すぎる → Disordered、臨界温度 → Coherent Phase
> - 評価が速すぎる → L-Growth収束、遅すぎる → Disordered、臨界評価時間 → V-Growth可能

---

## タスクの種類による T* の推定

タスクの「K値（相互依存度）」が T* を決める：

| タスク種類 | K値 | 推定 T* | 具体例 |
|----------|-----|---------|------|
| 感覚運動スキル | 低 | 即時〜数秒 | ピアノの打鍵、スポーツの反射 |
| 語彙・事実習得 | 低 | 数分〜数時間 | Duolingo、フラッシュカード |
| 概念理解 | 中 | 数時間〜数日 | 数学の定理、物理の法則 |
| フレームワーク変容 | 高 | 数週間〜数年 | PBD演奏者→作曲家 |
| パラダイムシフト | 非常に高 | 数年〜10年 | Bell Labs研究者の突破 |
| 文化・価値観変容 | 最高 | 世代単位 | 文明スケールのパラダイムシフト |

> **注意**: この表は定性的。T* の定量化は未解決問題（焼きなましのCooling Scheduleから類推できる可能性あり）。

## Cooling Schedule との対応

サイクル1（適応度景観）で発見した「焼きなまし × V-Growth」と接続:

| Cooling Schedule | 学習設計パターン | 向いている場面 |
|----------------|--------------|-------------|
| 指数冷却（最初は速い評価、徐々に遅く）| 早期フィードバック → 徐々に自律へ | K値が中程度のスキル習得 |
| 対数冷却（最初から遅い評価）| 長期的な評価遅延を最初から設定 | 深いパラダイム変容が目標 |
| 再加熱（停滞したら評価を止めて混乱再導入）| プラトーを検出したら意図的な Productive Failure | L-Growth の局所最適に捕まった時 |
| 適応的冷却（Phase Diagnos isでT*を動的調整）| 学習者の状態を見て評価サイクルを変える | 最高の精度（ただし師の診断能力が必要）|

## EdTechへの反論（L-Growth収束の構造的問題）

現代のEdTech（Duolingo, Khan Academy, etc.）はほぼ全て「即時フィードバック」に最適化されている。

**なぜか**: EdTechのビジネスモデルが「継続率・完了率・満足度」で評価されるから。これはAIのRLHFと同じ。短期的に満足感を高める「答えをすぐ教える」設計に収束する構造的引力がある。

**何を見落としているか**:
- Kが高い分野（数学的構造の深い理解、設計思想の体得）での学習
- フレームワーク変容型の学習（演奏者→作曲家）
- パラダイムシフトが必要な場面

**逆説**: EdTechが最も苦手なのが「最も教育効果が求められる」高K領域。Duolingoで外国語の「会話の文脈」は習得できない（高K）。四則演算は習得できる（低K）。

## 実践的応用

### 学習設計における診断フロー

```
Step 1: タスクのK値を推定する
  → 「このスキルの要素は互いに強く依存しているか？」
  → K低 = 各要素が独立 → 即時フィードバックでOK
  → K高 = 要素が絡み合う → T* を長めに設定

Step 2: 適切な評価サイクルを設計する
  → K低: 即時フィードバックツール（Duolingo、Anki）を使う
  → K中: 「数日後に自力で解いてみる」スパイラル学習
  → K高: 「混乱に住む」期間を意図的に設定。評価を保留する

Step 3: 停滞の診断
  → 進んでる感がある = L-Growth継続中（今の T* で続ける）
  → 「正しいが死んでいる」= V-Growth準備完了（T* を延長 or 再加熱）
  → 方向感がなくなった = Disordered（T* を短縮して小さな評価点を入れる）
```

### お兄ちゃんのPBD学習への具体的応用

現在のPBD学習は **フレームワーク変容型（K高）**:
- 演奏者型思考 → 作曲家型思考のV-Growth
- T* = 数週間〜数ヶ月単位の評価サイクルが適切

**具体的提案**:
- 「このパラメータを変えたら良くなった/悪くなった」= 即時評価 = L-Growthフィードバックループ
- 代わりに: 「今月の髪の動きを見て、生きてるか/死んでるかを1ヶ月後に振り返る」= 作曲家型評価サイクル
- PE（Permutation Entropy）測定 = 長期的な「動きの複雑さ」指標 = 作曲家型評価ツール
