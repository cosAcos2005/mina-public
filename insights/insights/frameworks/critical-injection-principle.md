## 一言で

量ではなく、受け手の臨界状態への精密な一撃が最大の変容を生む。

## 構造

SOCの洞察から：臨界状態にある系は、小さな摂動で最大の変化を起こす（雪崩）。
同じ摂動でも、非臨界状態の系では何も起きない。

これを認知・学習・創造の文脈に翻訳したもの：

```
二段階プロセス:
1. 臨界誘導: 受け手を「知覚的臨界状態」に導く（没入、ZPD境界、成熟した弟子）
2. 精密注入: 臨界状態の受け手に「最小の異化」を精密に当てる

結果:
・臨界状態 + 精密注入 → 雪崩（V-Growth・異化・洞察）
・非臨界状態 + 大量注入 → 無反応またはDisordered
```

### 臨界状態の種類

| タイプ | 定義 | 例 |
|--------|------|-----|
| 没入的臨界 | 物語世界のπ_priorが形成された状態 | 文学読書中の没入 |
| 成長臨界 | ZPD境界（独立能力の上限付近） | 学習の最適難易度点 |
| 成熟臨界 | π_prior蓄積による「崩れやすい状態」| 禅師が見極める弟子の成熟 |
| フロー臨界 | スキルと難易度のバランス点 | ゲームのフロー状態 |
| Frozen臨界 | π_prior過剰蓄積による「硬直」| 組織変革の窓（window of opportunity） |

### 精密注入の種類

| ドメイン | 精密注入の技法 |
|---------|-------------|
| 文学 | 前景化（foregrounding）——リズム・比喩・逆説 |
| 教育 | Socratic questioning——答えでなく問いで崩す |
| ゲーム | 新メカニクス投入——期待を裏切る設計 |
| 禅 | 公案——弟子の現在のπ_prior構造に当てる問い |
| 組織変革 | 外部衝撃ではなく内部Dissenter（異質者）の活用 |
| 音楽 | 解決（resolution）——テンション後の必然的収束 |

## 使いどころ

1. **「どれくらい変えれば良いか？」という問いへの応答**：量の前に「受け手が臨界状態にあるか」を問え
2. **PCE（Prior Collapse Engineering）の設計**：π_prior崩壊の量を増やす前に、臨界誘導の設計を優先する
3. **教育・コーチング**：「もっと強く教える」前に「学習者が成長臨界にあるか」を確認する
4. **ゲームデザイン**：難易度上昇の量より、フロー状態（臨界）の持続時間の最大化
5. **詩・小説の構造設計**：異化技法の「量」より、没入状態の構築に投資する

## 適用例

### 禅師の公案設計
禅師は公案の「強さ」を増やさない。弟子の成熟（=Frozen化した理解の蓄積）を待ち、
臨界状態に達した時、比較的小さな問いで雪崩を起こす。
「大きな公案」ではなく「正しいタイミングの公案」。

### Miall & Kuiken の実証的証拠
前景化は「文学的能力に依存しない」。なぜなら：
- 物語への没入（臨界誘導）は文学的能力と独立した普遍的神経状態
- 臨界状態に達した読者への精密な前景化は、誰でも応答する

### ZPDとの対応
ヴィゴツキーのZPD = 「成長臨界」の別名。
スキャフォールディング（足場）= 臨界誘導の技術。
「ちょうど届く難しさ」= 臨界状態への精密注入の定義。

## 限界

1. **臨界状態の測定困難**：受け手が「本当に臨界状態にあるか」は外から見えにくい
2. **「量がゼロでいい」ではない**：まず臨界誘導のためのある程度の蓄積（π_prior構築）が必要
3. **コンテキスト固有性**：何のπ_priorに対して臨界かが重要——「現実のπ_prior」と「物語世界のπ_prior」は別物（→ コンテキスト固有π_prior問題、pending）
4. **タイミングの非反復性**：同じ精密注入は二度目は機能しない（雪崩が起きた後、砂山は再臨界状態になるまで時間が必要）

## 過去のフレームワークとの関係

- **彫刻の原理**：制約を加えると構造が浮かぶ ← 加制約型の変容
- **臨界注入の原理**：臨界状態への精密注入で変容 ← 脱自動化型の変容
→ 二種類の変容アプローチ（彫刻=加制約 vs 臨界注入=脱自動化）として対になる

- **PCE**：Prior Collapse Engineering = 臨界注入の組織版実装
- **SND原理**：「戦略的に教えない」= 臨界状態の維持（誘導を解消しない）
- **SICサイクル**：臨界注入 = SICのStruggle+Incubationのトリガー設計
