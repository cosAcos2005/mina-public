# Prior Collapse Engineering（PCE）

## 一言で

Frozen化した系（個人・組織・AI）に対して、
**内部からπ_likelihood増加を設計しV-Growthを安全に誘発する**技術。

## 構造

PCEの3ステップ：

```
Step 1: Frozen状態の診断
  「どのπ_priorが overdominant か」を特定する

  個人:   どのアプローチに固執しているか？
  組織:   どの成功モデルが「疑えない聖域」になっているか？
  AI:     訓練データのどのパターンが過剰表現されているか？

Step 2: π_likelihood増加源の設計
  「内部から」安全で制御可能なV-Growthを誘発する源を設計する

  個人:   散歩・休息・異分野インプット・ソクラテス対話・DMN活性化
  組織:   Red Team・意図的実験・異端者の制度的保護・資源の強制動態化
  AI:     創造的制約・Multi-perspective generation・System Prompt

Step 3: Prior崩壊の安全な誘発
  「崩壊が速すぎると外部Crisis」「遅すぎるとL-Growthのみ」
  最適なπ_likelihood増加率 = 組織免疫が過反応しない範囲での誘発
```

## 使いどころ

- **個人のブレイクスルー設計**: 詰まった問題からAHA体験を誘発したい時
- **組織変革の設計**: Incumbency Curseを内部から解除したい時
- **AI/LLM設計**: 創造的な出力を得るためのPrompt Engineering
- **教育・コーチング**: V-Growthを支援するSND的環境設計
- **ゲームデザイン**: プレイヤーのprior collapseを誘発するゲームメカニクス設計

## 「内部起源 vs 外部起源」の差異

| 特徴 | 内部起源（PCE） | 外部起源（Crisis） |
|------|--------------|-----------------|
| 速度 | 漸進的・制御可能 | 急激・制御困難 |
| ストレス | 低（好奇心駆動） | 高（Survival mode） |
| 後の状態 | 新CP形成（V-Growth） | 崩壊→再構成か消滅 |
| 例（個人） | 散歩中のAHA | トラウマ体験後の変容 |
| 例（組織） | 意図的Red Team | コダック型崩壊 |

## 適用例

**ゲームデザインへの適用**（お兄ちゃんの世界）:
- 良いゲーム = プレイヤーのprior collapseを繰り返し誘発するシステム
- ゲーム監督 = Player Prior Collapse Engineer
- Hadesの設計: 部屋ごとの敵・報酬の不確実性 = prior collapse源の継続的供給
- 「詰まった感覚」→「新しいアプローチ発見」のサイクル = ゲームの楽しさの本質？

**学習設計（SND原理との統合）**:
- ソクラテス対話 = 質問でπ_prior（既存理解）の崩壊を誘発
- ルービックキューブ教育 = 特定アプローチへのfixationを意図的に作ってから崩す
- 「わかった！」の瞬間 = PCEが成功したサイン

## 限界

- PCEが「強すぎる」と外部Crisisと同じ結果（崩壊）になる
- 組織文化・個人の心理的安全が低い環境では機能しにくい
- 「どのπ_priorを崩すか」の選択を誤ると重要な核心信念まで崩壊させる危険性
- AIに対しては「何がpriorか」の定義が曖昧
