# 尤もらしさの罠（Likelihood Trap）

> [!abstract] 概要
> ニューラルシステムが「訓練データに多く見られるパターン（尤もらしさ）」を最大化することで、「実際の正しさ（物理整合性・事実整合性・論理整合性）」から外れる現象。そしてこの罠を避ける唯一の方法が「外部制約（prior）によるガイダンス」であることを示す原理。

---

## 🎯 一言で

**制約（prior）なしの生成は「最もありがちなもの」に収束する。それは正しくないことが多い。**

---

## 📋 構造

```
「尤もらしさ」の最大化
        ↓
訓練データの多数決パターンに収束
        ↓
Frozen方向へのドリフト（予測可能すぎる）
        ↓
物理的/事実的/論理的に「嘘」の出力
        ↓
ハルシネーション / ゴースト / 過学習
```

```
「外部制約（prior）」の注入
        ↓
可能性空間が制限される（彫刻の原理）
        ↓
Coherent Phase（制約内での尤もらしさ）
        ↓
物理的に正しく、かつ自然に見える出力
```

---

## 🗺️ 4つの顔（同一罠の異なるドメイン）

| ドメイン | 「尤もらしさ」の暴走 | 「外部制約」による救済 |
|---------|-----------------|-----------------|
| **ニューラルレンダリング** | 物理的に嘘のライティング・ゴースト | 3D物理情報（depth・法線・MV）でガイド |
| **言語モデル** | 訓練データに多い「それっぽい文章」= ハルシネーション | 事実データベース・RAG・ツール呼び出し |
| **金融予測モデル** | 過去の多数派パターンへの過学習 | マーケット構造制約・エコノミクス法則 |
| **ゲームNPC AI** | 訓練シナリオに多い行動に収束 | ゲームルール・物理エンジン・ナビゲーションメッシュ |
| **PBDシミュレーション** | 決定論的制約投影（pure prior projection） = 「機械的な動き」 | 確率的自由エネルギー最小化 = 熱ゆらぎ = 1/fノイズ |

---

## 💡 PBD × 尤もらしさの罠（最重要な接続）

> [!important] 髪物理エンジン開発への直結
> 
> **なぜPBDの動きは「機械的」に見えるか**、がこれで説明できる：
> 
> ```
> PBD        = 制約 C(x)=0 への決定論的投影 = pure prior projection
> 本物の布   = 熱ゆらぎ（kT）を含む確率的自由エネルギー最小化
>            = prior + stochastic noise = 1/fノイズが自然発生
> 
> 差分: 「posterior samplingの有無」
> ```
> 
> - 「死んだ動き」= posteriorサンプリングなしのpure prior投影
> - 「生きた動き」= prior + thermal noise = stochastic FE最小化 = 1/fノイズ
> 
> **結論**: PBDに1/fノイズを注入すると「生きた動き」になる仮説に、変分推論からの理論的根拠が付いた。確率的PBD（Stochastic XPBD）が正しいアーキテクチャである可能性が高い。

## 🔧 使いどころ

1. **ニューラルシステム設計**：「この生成プロセスは何に制約されているか？制約が弱い領域はどこか？」を問うことで設計の盲点を発見
2. **AIハルシネーション診断**：「どんな種類のpriorが欠けているか？」= どんな外部制約を追加すれば解決するか
3. **物理シミュ設計**：「確率的成分（thermal noise）を入れると1/f的自然さが生まれる」の理論的根拠として
4. **クリエイティブAI評価**：「これは尤もらしい（Likelihood Trap）か、それとも正しい（constraint-guided）か？」

## 🔗 彫刻の原理との関係

```
彫刻の原理（元）: 「制約が可能性空間を削ることで、正しい構造が浮かぶ」
                 → 量的な観点（どれくらい制約か）

尤もらしさの罠（深化）: 「制約がないと、尤もらしいが正しくないものが生まれる」
                       → 質的な観点（どんな種類の制約か、なぜ必要か）
```

**彫刻の原理の「なぜ制約が必要か」への答えが、尤もらしさの罠で与えられた。**

---

## 🔗 Coherent Phaseとの関係

```
制約なし生成 → 尤もらしさ最大化 → Frozen（予測可能すぎる）
制約あり生成 → posterior最適化 → Coherent Phase（制約内での生命感）
制約過剰     → 自由度なし       → Frozen（硬直）
```

尤もらしさの罠 = 「Frozenへの引力」。外部制約 = 「Coherent Phaseへの回帰」。

---

## ⚠️ 限界

- 「正しい制約」の設計が難しい（どんな物理情報が必要かを知っている必要がある）
- 計算コスト：制約を評価するコストが追加される
- 制約が強すぎると「創造性の欠如」（制約の逆U字：彫刻の原理のFrozen側）
