## 一言で

あらゆる「安定性と変化の問い」をHopfieldのエネルギー地形（谷と山の地形）として統一的に記述できる。

## 構造

Hopfield（1982）のエネルギー地形：
- **谷（Valley）** = 安定状態（引力子）。系はここに落ち着く
- **谷の深さ** = 安定性の強さ。深いほど動かしにくい
- **山（Ridge）** = 不安定状態。超えると別の谷へ
- **サドル点（Saddle）** = 谷と谷の間の峠。最も変化しやすい点

このメタファーを様々なドメインに適用：

```
状態    Frozen                  CP              Disordered
地形    深すぎる谷               浅い谷           平坦（谷なし）
動き    動けない                 動ける           どこへでも流れる
```

## 臨界注入との関係

**臨界注入の原理（今日蒸留）との接続**:
- 臨界状態（=受け手が変化しやすい状態）= サドル点（谷と谷の間）にいる状態
- 精密注入（=最小の一撃）= サドル点を超えて別の谷に入る最小エネルギー
- 「量より状態の問い」= 「量より、サドル点にいるかの問い」

```
臨界注入の最適条件:
1. 受け手をサドル点に誘導する（臨界誘導）
2. サドル点でほんの少しの力で別の谷に押し込む（精密注入）

失敗パターン:
- 深い谷の底にいる相手に大量の力を加える → 届かない or Disordered
```

## 使いどころ

1. **「変わらない」問題の診断**: 「深すぎる谷にいるか？サドル点にいないか？」
2. **変容設計**: 「まずサドル点に誘導する」戦略を先に考える
3. **REBUS強度の選択**: どのくらい「平坦化」が必要か？局所（詩的文脈）vs 大域（サイケデリック）
4. **「長続きするか？」の予測**: 変容後の新しい谷が「適度な深さ」か確認する（深すぎ → Frozen、浅すぎ → すぐ崩れる）

## 限界

1. **エネルギー関数の未定義**: 脳以外のドメインで「エネルギー」が何かが不明確
2. **地形の高次元性**: 実際の意味空間・組織空間は何百次元。2D地形のメタファーは過剰単純化
3. **不動点思考との重複**: 圏論的「不動点」と「エネルギー地形の谷」は同じものを異なる言語で記述している。数学的関係は未確立
4. **動的変化**: 実際のエネルギー地形自体が時間とともに変化する（Fristonの生成モデルも更新される）

## 他のフレームワークとの関係

- **臨界注入の原理**: エネルギー地形の「サドル点通過」として具体化
- **不動点思考**: 「谷」= 「不動点」の物理的実装
- **彫刻の原理**: 加制約（谷を深くする）vs 脱制約（谷を浅くする/平坦化）の二方向
- **SICサイクル**: Struggle（サドル点に向かう移動）→ Incubation（サドル点付近の不安定な滞留）→ Crystallization（新しい谷への着地）
